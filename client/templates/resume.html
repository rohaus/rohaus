Phone Tag November 2013 – December 2013

Phone tag is a geolocation-based game transforms their community into an arena. Gameplay is similar to laser tag and players use their smartphone to track and tag players. I collaborated in a four-person team to build and deploy the first major version of this open-source, full-stack, mobile app. My general role was in front-end development and implementation of sockets for real-time communication. In the near future I plan on working adding more game features and another mode of gameplay.

This app was built using Backbone.js, Node.js, Express.js, Socket.io, PhoneGap (iOS + Android), Require.js, and Hammer.js.

Initially, the mobile app had moments of notably slow performance, so I removed jQuery mobile and balanced game logic between client-side and server-side to improve performance.

Barter - November 2013

Full-stack bartering web interface that allows users to geospatially search for items available in their community. Barter uses Facebook authentication and has a built-in messaging and trade request system to streamline the bartering process. The interface was built on top of Google Maps with utmost consideration for the user’s experience. The first major version has been deployed and beta tested with a small group of users. In the near future I plan on optimizing the website for mobile.

Barter was built with a MEAN stack (MongoDB, Express.js, Angular.js, and Node.js) and Passport.js.

Google Maps does not natively support marker clustering, which made searching the map difficult. I implemented instant search and two forms of marker clustering to improve users usability and search ability.
